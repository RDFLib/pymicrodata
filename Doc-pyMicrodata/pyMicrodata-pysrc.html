  2  """ 
  3   
  4  This module implements the microdata->RDF algorithm, as documented by the U{W3C Semantic Web Interest Group 
  5  Note<http://www.w3.org/TR/2012/NOTE-microdata-rdf-20120308/>}. 
  6   
  7  The module can be used via a stand-alone script (an example is part of the distribution) or bound to a CGI script as a Web Service. An example CGI script is also added to the distribution. Both the local script and the distribution may have to be adapted to local circumstances. 
  8   
  9  (Simple) Usage 
 10  ============== 
 11  From a Python file, expecting a Turtle output:: 
 12   from pyMicrodata import pyMicrodata 
 13   print pyMicrodata().rdf_from_source('filename') 
 14  Other output formats are also possible. E.g., to produce RDF/XML output, one could use:: 
 15   from pyMicrodata import pyMicrodata 
 16   print pyMicrodata().rdf_from_source('filename', outputFormat='pretty-xml') 
 17  It is also possible to embed an RDFa processing. Eg, using:: 
 18   from pyMicrodata import pyMicrodata 
 19   graph = pyMicrodata().graph_from_source('filename') 
 20  returns an RDFLib.Graph object instead of a serialization thereof. See the the description of the 
 21  L{pyMicrodata class<pyMicrodata.pyMicrodata>} for further possible entry points details. 
 22   
 23  There is also, as part of this module, a L{separate entry for CGI calls<processURI>}. 
 24   
 25  Return formats 
 26  -------------- 
 27   
 28  By default, the output format for the graph is RDF/XML. At present, the following formats are also available (with the corresponding key to be used in the package entry points): 
 29   
 30   - "xml": U{RDF/XML<http://www.w3.org/TR/rdf-syntax-grammar/>} 
 31   - "turtle": U{Turtle<http://www.w3.org/TR/turtle/>} (default) 
 32   - "nt": U{N-triple<http://www.w3.org/TR/rdf-testcases/#ntriples>} 
 33   - "json": U{JSON-LD<http://json-ld.org/spec/latest/json-ld-syntax/>} 
 34
